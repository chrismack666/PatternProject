Используемые паттерны проектирования:
1. Singleton. Используем в классе Logger.java, чтобы все данные при логировании записывались в один файл, и
не затирались после каждого создания нового объекта.
2. Factory. Используется в классе FactoryVehicle.java для получения соответствующего наследника класса (Vehicle)
в зависимости от входных параметров (Bicycle, Motorbike, Car, Truck). Для каждого транпортного средства логируем
количество его колес.
3. Adapter. Испольуется в классе CarAdapter.java для конвертирования двух несовместимых объектов. У разных автомобилей
есть различные виды ходовых огней, в данном примере рассматриваем автомобили, у которых в качестве базового света есть
дневные ходовые огни CarRunningLigth.java или габаритные огни CarParkingLight.java. С помощью класса CarAdapter.java
возможно вызывать метод ligthsOn() для любого из этих двух типов автомобилей.
